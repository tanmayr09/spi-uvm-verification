⚠ Key architectural difference vs SPIMaster: The slave has no internal clock generator — it uses the master's SCLK directly. The RX and TX cycle-count blocks are clocked by i_sclk edges, not the FPGA clock. This means two clock domains exist inside this module: w_Clk domain (FPGA) and w_sclk domain (SPI). The registered i_mosi/i_sclk/i_ss signals cross from the SPI domain back to the FPGA domain safely.
🔒 SPI Input Registration (CDC)
Registers all SPI bus inputs on FPGA clock to prevent metastability.

r_mosi ← i_mosi  |  r_sclk ← i_sclk  |  r_ss ← i_ss (reset=1)

Critical: r_ss resets to 1'b1 (deasserted) not 0 — slave starts deselected.
This was one of the RTL bugs that was already fixed in your project.
⏱ SCLK Polarity Adjust
assign w_sclk = w_CPOL ? !r_sclk : r_sclk

Adjusts registered SCLK for CPOL setting.
w_sclk is used as the clock edge for all generate blocks.

CPOL=0: w_sclk = r_sclk (no inversion)
CPOL=1: w_sclk = !r_sclk (inverted)
🔴 SS Detector + Busy
r_ss directly controls busy state.

o_busy = !r_ss (combinational)

r_ss=0 → busy=1 (transfer active)
r_ss=1 → busy=0 (idle)

No FSM needed — SS itself IS the state signal for the slave.
📥 TX Shift Register
r_tx_byte latches i_tx_byte when
i_tx_ready=1 AND r_ss=1 (must preload BEFORE transfer)

w_tx_byte = r_tx_byte << r_cycle_count
(combinational shift — always shows correct bit)

w_miso = w_tx_byte[7] (MSB first)

r_cycle_count is clocked by w_sclk edges — not FPGA clock!
📤 RX Shift Register
Clocked by posedge w_sclk (Mode 0/2)
or negedge w_sclk (Mode 1/3)

r_rx_byte ← {r_rx_byte[6:0], r_mosi}
Shifts in MSB first from registered MOSI

Uses generate block to select edge at compile time based on MODE parameter — not runtime logic.
➡ MISO Output
assign o_miso = r_ss ? 'z : w_miso

r_ss=1 (idle): MISO = high-Z (bus released)
r_ss=0 (active): MISO = w_miso (drives data)

This is the OPPOSITE polarity from MOSI in master — slave releases when NOT selected.
⚙ Generate Blocks — Compile-time Mode Selection
Cycle counter generate:
MODE 0/2: negedge w_sclk → increment r_cycle_count
MODE 1/3: posedge w_sclk → increment r_cycle_count
(reset on posedge r_ss)
RX shift generate:
MODE 0/2: posedge w_sclk → shift r_rx_byte
MODE 1/3: negedge w_sclk → shift r_rx_byte
(reset on negedge i_Rst_n)
✅ Output Stage — o_rx_byte + o_byte_ready
Triggered when: r_ss=1 AND o_busy=1 (SS just went HIGH = transfer just ended)

o_rx_byte ← r_rx_byte  |  o_byte_ready ← 1 for exactly 1 FPGA clock cycle

Next cycle: o_byte_ready ← 0

This is clocked on FPGA clock (w_Clk), not SPI clock — safe to read from FPGA logic.
⚙ SPI Mode Decoder (CPOL / CPHA) — same as master
MODE 0: CPOL=0, CPHA=0 — w_sclk idles LOW, sample rising ↑
MODE 1: CPOL=0, CPHA=1 — w_sclk idles LOW, sample falling ↓
MODE 2: CPOL=1, CPHA=0 — w_sclk idles HIGH, sample falling ↓
MODE 3: CPOL=1, CPHA=1 — w_sclk idles HIGH, sample rising ↑
